Test Case Writing Guide

Good test cases are:

* Clear
* Repeatable
* Independent
* Easy to understand
* Focused on one objective

Best Practices

* Test one feature per test case.
* Write concise but complete steps.
* Define expected results clearly.
* Include required test data.
* Avoid assumptions.
* Keep test cases up to date as features change.

A well-written test case should allow another tester to execute it without additional explanation.
